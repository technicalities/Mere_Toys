Assembly language

Syntax of assembly language

An assembly language program is a sequence of lines of text. Each line is a complete, separate statement. There are no nested structures, such as begin-end blocks or while loops, that begin on one line and continue on to another.

An assembly language statement may be entirely blank. Blank lines are useful for breaking a program up into sections of code.

The semicolon character (;) starts a comment, which continues until the end of the line. Thus the assembler will ignore everything that comes after a ;. Comments may appear in the last part of a statement, or they may fill the entire line.

Every statement consists of three fields. The first field must begin in the first character of a line, and the fields are separated by white space characters (spaces and/or tabs). One or more of the fields may be empty.

Statements that generate code

Some assembly language statements generate one or more words of code to be placed in memory. These include instructions and data definitions.

Instructions

Data definitions

name data constant

The data statement generates a word with the specified value, which may be specified as a signed decimal integer (e.g. -23) or as a 4-digit hexadecimal word (e.g. $0a4c). The name field is optional; if present, the name is equivalent to the address of the word.
